Spielregeln:

Vor den Spielern liegt die Insel Catan. Die Aufgabe ist es, diese rundenweise Insel zu besiedeln. Dabei erhalten sie Punkte und wer als erster 10 Punkte erhalten hat, hat gewonnen.


Es gibt 6 verschiedene Landschaftentypen auf der Insel die verschiedene Rohstoffe erzeugen: 
- Wald erzeugt Holz, 
- Ackerland erzeugt Getreide, 
- Hügelland erzeugt Lehm, 
- Weideland erzeugt Wolle, 
- Gebirge erzeugt Erz und 
- Wüste erzeugt keine Erträge.
18 rechteckige, kreisförmig angeorten Landschaftenfeldern bilden das Spielfeld umgeben von Meer.
Daraus ergibt sie ein solches Muster:   
         ___ ___
        | x | x |
     ___|___|___|___ 
    | x | x | x | x |
 ___|___|___|___|___|___
| x | x | x | x | x | x |
|___|___|___|___|___|___|
    | x | x | x | x |
    |___|___|___|___|
        | x | x |
        |___|___|

An jedem "X" befindet sich eine Landschaftfeld.
Auf einen zufälligen Feld befindet sich zudem der Räuber.

Die Spieler können nun an den "Ecken" der Landschaftsfelder Siedlungen plazieren. Z.B.:

         ___ ___
        |   |   |
     ___|___x___|___ 
    |   |   |   |   |
 ___|___|___|___|___x___
|   |   |   |   |   |   |
|___|___x___|___|___|___|
    |   |   |   |   |
    |___|___|___|___|
        |   |   |
        |___|___|

Außerdem können, angrenzent zu einer Siedlung im Besitz des Spielers, Straßen auf den "Kanter der Landschaftsfelder plaziert werden. Beispeil (x = Siedlung, --- = eine Straße)

         ___ ___
        |   |   |
     ___|___|___|___ 
    |   |   |   |   |
 ___|___x---|___|___|___
|   |   |   |   |   |   |
|___|___|___|___|___|___|
    |   |   |   |   |
    |___|___|___|___|
        |   |   |
        |___|___|
        
An dem manchen "Kanten" zum Meer befinden sich zudem Häfen mit besondern Handels Optionen. Dieser eröffenen Spielern verbesserte Handelsangbot mit der Bank wenn sie eine Straße zu Ihnen bauen.
Beispiel: x = Siedlung, --- = eine Straße, h = Hafen, H = nutzbarer Hafen
         _h_ ___
        |   |   |
     ___|___|___|___ 
    |   |   |   |   h
 ___|___|___|___|___|___
h   |   |   |   |   |   |
|___|___|___|___|___|___|
    h   |   |   |   |
    |___|___x---|-H-|
        |   |   | 
        |___|___|


Den Landschaftsfeldern sind 6 unterschiedliche Wahrscheinlichkeiten, von sehr häufig bis sehr selten, zugeortnet die die Spieler den Felder, durch farbliche makierung, ablesen können.
Zu beginn jedes Zuges werden zufällig Landschaftsfelder ausgewählt die Rohstoffe erschütten. Felder mit einer höherer Wahrscheinlichkeit geben natürlich im Durchschnitt häufiger Rohstoffe.
Ein Spieler erhält Rohstoffe von einem Landschaftsfelder falls er eine Siedlung hat die an ein solches grenzt. Zum Beispiel kann die Siedlung "X" von allen mit "r" makierten Feldern Rohstoffe erhalten:
         ___ ___
        |   |   |
     ___|___|___|___ 
    |   | r | r |   |
 ___|___|___x___|___|___
|   |   | r | r |   |   |
|___|___|___|___|___|___|
    |   |   |   |   |
    |___|___|___|___|
        |   |   |
        |___|___|
        
Zur Beginn der Partie dürfen die Spieler jeweils 2 Siedlung und 2 Straßen plazieren. Die Spielfieguren der Spieler unterscheiden sich dabei farbig.
Danach beginnt jede Runde mit der Rohstoffauschüttung, wie oben beschrieben, für alle Spieler.
Sonderfall: Mit zweit größter Häufigkeit erhält kein Spieler Rohstoffe sondern der Räuber wird aktiv. Alle Spieler, die mehr als 7 Rohstoffe besitzen, verlieren die Hälfte ihrer Rohstoffe. Bei ungeraden Zahlen wird zu Gunsten des betroffenen Spielers abgerundet (wer z. B. 9 Rohstoffe hat, verliernt 4 davon). Der Räuber raubt eine zufällige der, zu dem Landschaftsfeld auf dem er sich befindet, angrenzende Siedlung aus. Das bedeutet das der Besitzer dieser Siedlung einen weitere zufälligen Rohstoff verliert und dieser an den akitven Spieler übertragen wird.

Der aktive Spieler darf dann Mitspielern und der Bank handeln, Entwicklungskarten spielen und bauen.

Mit Mitspielern können alle Rohstoffe und Karten in belieben Kombinationen handeln, solange beide diesem Angebot zustimmen.
Mit der Bank können Rohstoffe 4 zu 1 gehandelt werden, d.h. 4 gleiche Rohstoffe für einen belieben anderen (z.B. 4 Holz für ein Lehm).
An dem manchen "Kanten" zum Meer befinden sich zudem Häfen mit besondern Handels Optionen. Dieser eröffenen Spielern verbesserte Handelsangbot mit der Bank wenn sie eine Straße zu Ihnen bauen (z.B. 3 zu 1 für alle Rohstoffe, 2:1 für Weizen, etc.)
Beispiel: x = Siedlung, --- = eine Straße, h = Hafen, H = nutzbarer Hafen
         _h_ ___
        |   |   |
     ___|___|___|___ 
    |   |   |   |   h
 ___|___|___|___|___|___
h   |   |   |   |   |   |
|___|___|___|___|___|___|
    h   |   |   |   |
    |___|___x---|-H-|
        |   |   | 
        |___|___|
        
Außerdem kann er eine zufällige Entwickungskarten von der Bank für ein Erz + ein Wolle + ein Getreide kaufen. Es gibt drei Arten von Entwicklungskarten: Ritter, Fortschritt und Siegpunkte.
Ritter: Wer eine Ritterkarte ausspielt, versetzt den Räuber auf ein zufälliges anders Feld. Wer mindestens 3 Ritterkarten und mehr Ritterkarten als aller anderen ausgespielt hat, erhält zu dem die Sonderkarte "Rittermacht", diese ist 2 Punkte wert.
Erfindung: Der Spieler erhält 2 zufällige Rohstoffe.
Monopol: Der Spieler erhällt alle Rohstoffe eines der Karte genanntes Typs von seinen Mitspielern.  Wer keinen solchen Rohstoff besitzt, muss auch nichts abgeben.
Siegpunkte: Diese geben einen Punkt.

Der akitver Spieler darf zudem bauen. 
Für einen Lehm und ein Holz kann der eine Straße nach den oben gannten Regel platziern. 
Der Spieler der die längste zusammenhängende Straße besitzt erhält den Bonus "Längste Handelsstraße". Dieser ist 2 Siegpunkte wert
Weitere Siedlungen kosten ein Lehm, ein Holz, ein Wolle und ein Getreide. 
Voraussetzung hierfür ist, dass eine der eigenen Straßen dort hin führt und die neue Siedlung ist mindestens 2 "Kanten" weit entfernt von jeder anderen. Jede Siedlung ist 1 Punkt wert.
(x = Siedlung, --- = eine Straße, b = geeigenter Bauplatz für eine neue Siedlung, S = Siedlung ander Spieler)
         ___ ___
        |   |   |
     ___|___|___|___ 
    |   |   |   |   |
 ___|---|---x---|---b___
|   |   |   |   |   |   |
|___S___|___|___|___|___|
    |   |   |   |   |
    |___|___S___|___|
        |   |   |
        |___|___|


Eine Siedlung kann zu einer Stadt ausgebaut werden, für 3 x Erz + 2 x Getreide. 
Diese ist 2 Siegpunkte wert und angrenzende Landfelder geben jedes mal zwei Rohstoffe anstatt von einem. 

Darüber hinaus kann er aktive Spieler Entwicklungskarten in seinem Besitz spielen. 

Der Spieler der zuerst 10 Spiegpunkte erreicht hat gewonnen.



Seitenfunktionen:

Spieler können sich über die Startseite registieren bzw. einloggen.
Sie haben die Möglichkeit einer laufenden Partie beizutreten, eine neue Partie mit einer festen Spieleranzahl zu erstellen oder eine, auf ihrem Account gespeichtere, Partie zu laden.

Auf dem eigenlichen Spielfälche wird Ihnem, neben dem Spielfeld und den anderen Spielfunktionen, noch weitere Dinge angezeigt:
- Der Status der Partie: Name des Spielers der am Zug ist, Rundenanzahl, die Möglichkeit das Spiel zu speichern.
- Ihr eigener Status: Name des Spielers, am Zug?, meine Farbe, Punktanzahl, Straßen, in Besitz, Verbundne Rohstoffe, Anzahl Siedlungen, Anzahl Städte, Rohstoffe und Anzahl im Besitz
- Ein Chat-Fenster ermöglich es den Spielern, mit Hilfe öffentlicher und privater Gespräche, verhandlungen zu führen.
- Eine Zusammenfassung der Spielregeln in einem Tutorial und der Baukosten

Außerdem gibt es jederzeit die Möglichkeit auf einen Impressum, den vollständigen Spielregeln und auflisten aller Baukosten, sowie die Siete des originalen Brettspiels als auch einer Version der Seite auf Englisch zuzugreifen.

Vor den Spielern liegt die Insel Catan. Sie besteht aus viereckigen Landfeldern, mit Meer rundherum. Die Aufgabe ist es, diese Insel zu besiedeln.

Spieler eröffenen über eine Startseite eine Partie oder tretten einer erstellenten Partie bei. Dabei ist es möglich sowas mit menschlichen als auch mit Computer-Gegner zuspielen.
Es besteht ausserdem die Möglichkeit eine laufende Partie zu speichern und wieder zu laden.
Letztere reagieren auf das Spielverhalten der anderen, passen so ihres an und veränderen dementsprechend ihre Handelsangebote.

Alle wichtig Informationen und Karten des Spielers werden angezeigt.

Es gibt zwei Startvarieanten: 
- für Einsteiger, welches ein festes Spielfeld hat und die Spielregeln am Anfang der Partie anzeigt und 
- für Fortgeschrittene welches eine zufällig generierte Spielfeld bietet.

Es gibt 6 verschiedene Landschaften die verschiedene Rohstoffe erzeugen: 
- Wald erzeugt Holz, 
- Ackerland erzeugt Getreide, 
- Hügelland erzeugt Lehm, 
- Weideland erzeugt Wolle, 
- Gebirge erzeugt Erz und 
- Wüste erzeugt keine Erträge. 
Landfeldern erhalten zudem jeweils eine Nummer. 

An dem Kanten zum Meer befinden sich Häfen mit besondern Handels Optionen. 
Siedlungen/Städte können auf den freien Ecken und Strassen auf freien Kanten der Felder platziert werden.
Die Ecken und Kanten veränderen passend die ihre Farbe, je nach dem zu welchem Spieler sie gehören.
Ein Spieler kann die Rohstoffe von Feldern "ernten" die an eine Siedlung/Stadt von ihm grenzen.

Am Anfang der Partie platziert jeder Spieler 2 Straßen und 2 Siedlungen auf dem Spielfeld. 
Die Spielfieguren der Spieler unterscheiden sich dabei farbig.

Zu beginn jedes Zuges werden zwei Zufallszahlen generiert und zusammen gerechenet. 
Jedes Feld die die Nummer dieses "Würfelwurfs" hat liefert den entsprechenden Rohstoff. 

Nun kann der aktive Spieler mit Mitspielern handeln. 
Alle Rohstoffe und Karten in belieben Kombinationen handeln, solange beide diesem Angebot zustimmen.
Ein Chat-Fenster ermöglich es den Spielern, mit Hilfe öffentlicher und privater Gespräche, verhandlungen zu führen.  
Außerdem kann der aktive Spieler mit der Bank handeln, Entwicklungskarten kaufen und besondere Handelsangebote der Häfen nutzen falls sie mithilfe einer Strasse angebunden hat. 

Der akitver Spieler darf zudem bauen. 
Für einen Lehm und ein Holz kann der eine Straße platziern. 
Der Spieler der die längste zusammenhängende Straße besitzt erhält den Bonus "Längste Handelsstraße".
Neue Siedlungen kosten ein Lehm, ein Holz, ein Wolle und ein Getreide. 
Voraussetzung hierfür ist, dass eine der eigenen Straßen dort hin führt und die nächste Siedlung ist mindestens 2 Kreuzungen weit entfernt von jeder anderen. 
Eine Siedlung kann zu einer Stadt ausgebaut werden. 
Diese ist mehr Siegpunkte wert und angrenzende Landfelder geben jedes mal zwei Rohstoffe anstatt von einem. 

Darüber hinaus kann er aktive Spieler Entwicklungskarten in seinem Besitz spielen. 
Diese geben entweder Siegpunkte oder sind Fortschrittkarten, welche jeweils beim ausspielen den auf ihnen beschrieben Effekt ausführen.

Der Spieler der zuerst 10 Spiegpunkte erreicht hat gewonnen.

=== Feedback Alpers, 2016-04-05 ===
===================================

Die Idee ist leider nicht gut, weil Sie "ein interaktives Programm" entwickeln sollen. Das bedeutet, dass es Elemente geben soll, die Nutzer anwählen können und/oder Bereiche, in denen sie etwas eintragen oder auswählen können. Es soll im Hintergrund ein Programm aktiv sein, das eine gewisse Funktionalität umsetzt und mit dem NutzerInnen interagieren können.

Im Gegensatz dazu bieten soziale Netzwerke im Grunde nicht mehr an als jedes Mail-Programm: Texte an beliebige Empfänger verschicken und empfangen, sowie beliebige Anhänge einfügen. Die Funktionalitäten, die dabei sicher gestellt werden müssen haben vorrangig etwas mit der Administration von Servern und Datentransfers zu tun. Es gibt nur zwei Unterschiede zwischen sozialem Netzwerk und E-Mail-Client: Der eine ist die Darstellung (also das Design und damit etwas, was nicht Teil dieses Kurses ist) und das andere ist die Tatsache, dass soziale Netzwerke vorrangig dafür entwickelt werden, um mittels Methoden des Big Data massenhaft Daten über NutzerInnen zu gewinnen und diese zu Gewinnzwecken zu verwenden.

Versuchen Sie deshalb, entsprechend der nächsten Aufgabe ein "Siedler von Catan" als Projektidee zu beschreiben. (Wenn Sie das Spiel nicht kennen, schauen Sie sich im Netz um, Sie werden dort eine Vielzahl an Videos und Beschreibungen finden.)

=== Feedback Alpers, 2016-05-31 ===
===================================

Leider ist Ihre Beschreibung noch deutlich zu nahe am Regelwerk des Brettspiels. Eine Beschreibung sollte in den ersten Zeilen Lust darauf machen, mehr zu erfahren.

Eine Zusammenfassung sollte wie gesagt das Interesse am Spiel wecken, also müssen Sie darin in sehr knapper Form den Kern des Spiels zusammenfassen. Und der hat mit dem Zählen von Punkte oder der Anzahl von Spielfeldern nichts zu tun. Das selbe gilt für die folgenden Sätze, in denen Sie angeben, wie Spieler wie viele Punkte bekommen können.

Denken Sie bitte auch daran, dass es nicht um eine 1:1-Umsetzung des Brettspiels selbst geht. Hier denke ich beispielsweise an die Spielerzahl: Es sollte möglich sein, dass ein Spieler das Spiel alleine spielen kann (Stichwort: Computergegner), aber schauen Sie auch, welche Möglichkeiten es gibt, um das Spiel für mehr als 5 Spieler anzupassen.

Danach sollte die Beschreibung alle für Spieler relevanten Teile des Spiels beinhalten. Hier sind Sie leider auch zu nahe am Brettspiel: Wenn Sie die Spielfelder mit den Zahlen 2 bis 11 nummerieren, dann beschränken Sie sich unnötig. Beim Brettspiel gibt es dagegen kaum eine andere Möglichkeit. Und umgekehrt ist es schwierig, mit den Mitteln, die Sie bislang kennen gelernt haben, Sechsecke zu programmieren. Quadrate sind dagegen ganz einfach.

Die Projektbeschreibung ist also im Kern eine Inhaltsangabe. Der Unterschied gegenüber dem Deutschunterricht ist der, dass Sie ein Computerprogramm zusammenfassen müssen, das Sie selbst konzipieren sollen und für das einige Vorgaben einzuhalten sind.

Für die Umsetzung in ein Computerspiel müssen Sie jetzt genau herausarbeiten, was für Interaktionsmöglichkeiten es geben soll. Die sind wiederum die Basis für die interaktiven unter den strukturellen Elementen. Und machen Sie hier bitte nicht den Fehler, die Elemente des Brettspiels als Elemente des Computerspiels umzusetzen.

Hier noch ein Absatz zum Thema Interaktion: Die Schaltflächen, die NutzerInnen verwenden können sind nur die bedienbaren Elemente eines Computerprogramms. Interaktion ist wesentlich mehr als das. Denken Sie beispielsweise daran, wie eine Pokerrunde gespielt wird. Dabei spielt es nämlich eine Rolle, welche Karten abgeworfen wurden. Die Spieler (und bei einem Computerprogramm der Computer) analysieren kontinuierlich das Verhalten der anderen Spieler und ändern Ihren Spielstil in Abhängigkeit davon. Und genau das müssen Sie auch als Teil Ihres Projekts ausarbeiten und in die Projektbeschreibung integrieren.

Hier die zweite Schwierigkeit: Sie müssen sich überlegen, was Sie alles streichen und was sie alles ergänzen müssen, damit es ein eigenständiges webbasiertes Computerspiel wird.

Sie müssen bei der Projektkonzeption also insbesondere an Dinge denken, die in einem Brettspiel nicht vorkommen. Ein Log-In oder eine Highscore-Liste wären nur zwei von vielen Möglichkeiten, die hinzukommen. Die direkte Unterhaltung zwischen Spielern muss dagegen im Bedarfsfall anders gelöst werden. Und Aufgaben wie das Aufnehmen einer Karte, das verschieben eines Elements auf dem Spielbrett fallen häufig bei Computerspielen unter den Tisch.

Denken Sie bitte auch daran, dass Änderungen in einem Teil des Projekts häufig auch Änderungen in einem anderen Teil des Projekts nach sich ziehen.

Und arbeiten Sie bitte mit Überschriften, aus denen erkennbar ist, was die nächsten Zeilen beschreiben. (Stichwort: Regeln, Strukturen, ...)

== I ==
Die Beschreibung ist weitgehend gut, allerdings orientiert sie sich noch zu sehr am Brettspiel bzw. an den Beschränkungen, die ein Brettspiel nötig macht.

=== Zusammenfassung ===

Zurzeit haben Sie eine Projektbeschreibung, bei der kaum erkennbar ist, wie sich die einzelnen Teile in der Struktur wiederfinden.
Außerdem ist erkennbar, dass Sie bislang kaum etwas konkret ausgearbeitet haben, das nicht auch so oder nahezu in dieser Form im Brettspiel steckt.
Sie haben zwar an der Struktur gearbeitet, aber aufgrund der Art, wie Sie das getan haben ist kaum erkennbar, wo welche Änderungen durchgeführt wurden.
Und obwohl Sie Gruppen definiert haben, haben Sie noch keine Zeile Quellcode entwickelt.

Deshalb nochmal der ganz klare Hinweis:

Ein Softwareprojekt wird heute praktisch immer so entwickelt, dass ständig an allen Teilen des Projekts Änderungen erfolgen. Ein Ablauf wie beim sogenannten Wasserfallmodell, bei dem immer nur an einer Phase gearbeitet wird, nach deren Abschluss die nächste Phase begonnen wird gibt es heute nur noch im Ingenieursbereich. In der Medieninformatik ist dieser Ansatz praktisch nicht existent. Sie müssen sich daran gewöhnen, immer wieder zu früheren Stellen eines Projekts zurück zu kehren, dort Änderungen auszuführen und auch die aus diesen Änderungen resultierenden Änderungen an anderer Stelle eines Projekts umzusetzen.
